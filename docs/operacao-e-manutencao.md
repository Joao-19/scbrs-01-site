# Operação e Manutenção

## Como rodar o projeto
Na raiz do repositório:

```powershell
npm install
npm run dev
```

## Comandos principais
- `npm run dev`
- `npm run build`
- `npm run preview`

## Responsabilidades por pasta
- `frontend/`: código da aplicação
- `roadmaps/`: planejamento e histórico de fases
- `ai_context/`: contexto técnico para IA (inglês)
- `docs/`: regras de negócio para humanos (português)

## Boas práticas de manutenção
- Atualizar primeiro conteúdo, depois visual.
- Validar build após cada alteração relevante.
- Manter consistência entre copy e proposta real da comunidade.

## Evoluções futuras possíveis
- Backend para dados dinâmicos (se aprovado).
- Integração automática de indicadores comunitários.
- Biblioteca de componentes para acelerar futuras páginas.
